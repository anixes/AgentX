# Phase 21: System-Wide Uncertainty Propagation

## Overview

As the final hardening step before transitioning to the Phase 11 Planning Layer, the AgentX reliability engine has been upgraded to track and propagate **uncertainty** at a system-wide level. This prevents silent error accumulation and hallucination drift by treating uncertainty as a tangible risk metric that compounds over time and triggers safety escalations.

## Core Features

### 1. Extensible Uncertainty Scoring
The evaluation pipeline (`agentx.decision.evaluator`) now computes a granular `uncertainty_score` (float between 0.0 and 1.0) for every evaluated task step.

Factors contributing to uncertainty:
* **Disagreement:** Conflicting verdicts among evaluators (+0.3).
* **Soft Vetoes:** When a highly reliable evaluator casts a veto, but not strong enough to immediately fail the task (+0.4).
* **Confidence Delta:** Inversely proportional to the evaluator's confidence.
* **Veto Triggered:** Hard veto occurrences push the uncertainty score (+0.5).

### 2. Task-Level Uncertainty Tracking
* Uncertainty is accumulated on a per-task basis.
* Every retry loop iteration or evaluation step (`run_mission` in `agentx.py`) adds the current `uncertainty_score` to the running `task_uncertainty_score`.
* **Decay Factor:** To prevent uncertainty from endlessly exploding over long but successful chains, `task_uncertainty_score` is decayed per step (multiplied by `0.9`).

### 3. Risk-Aware Escalation Thresholds
* `MAX_TASK_UNCERTAINTY` is set to `0.8`.
* If a task's accumulated uncertainty exceeds this threshold, the system immediately **stops execution**, logs a `SYSTEM_UNCERTAINTY_EXCEEDED` event, and escalates to a human via `ASK`.

### 4. Hierarchical Execution Awareness (Multi-Step Uncertainty)
* In `hierarchical_execution.py` (formerly skill_composer.py), when executing a multi-step `Hierarchical Execution` chain, uncertainty is evaluated per sub-objective.
* If **multiple steps** (>= 2) yield an `UNCERTAIN` evaluation or have high uncertainty scores (> 0.4), the entire chain halts early, emitting `CHAIN_UNCERTAINTY_EXCEEDED` and falling back to safety defaults.

### 5. Metrics & Database Integration
* `metrics.py` schema has been updated to include `uncertainty_score` in the `decision_metrics` table.
* Backwards compatibility via `ALTER TABLE` is handled seamlessly.
* `get_metrics()` now calculates and exposes `avg_uncertainty_per_task` and `uncertainty_to_failure_ratio`.

## Readiness for Planning Layer
The AgentX Strategy Selection Module is now fully probabilistically aware and capable of guarding against deep hallucination loops, making it robust enough to handle the complex, non-deterministic chains generated by the Phase 11 Planning Orchestrator.
